## 方案一：请求超时（15s/proxy）

### 原理

利用 `AbortController` + `setTimeout` 组合，给每次代理请求设置独立超时。

### 实现

在 `fetchArticleFromUrl` 中，为每个代理请求创建独立的超时 controller，与上层传入的用户取消 signal 合并（`AbortSignal.any`，或手动 race）：

```js
async function fetchWithTimeout(url, signal, timeoutMs = 15000) {
  const timer = new AbortController()
  const id = setTimeout(() => timer.abort(), timeoutMs)
  try {
    return await fetch(url, {
      signal: signal
        ? AbortSignal.any([signal, timer.signal])
        : timer.signal,
    })
  } finally {
    clearTimeout(id)
  }
}
```

> `AbortSignal.any` 在较新浏览器支持，需做兼容降级（手动 race）。

## 方案二：标题清洗

### 原理

Readability 的 `article.title` 常包含站点名，如：
- `"How I Built This | Medium"`
- `"My Article — Substack"`
- `"Post Title - The Atlantic"`

用正则去掉分隔符及其后内容，但要避免误伤标题本身带有破折号的情况（只去最后一个分隔符）。

### 实现

```js
function cleanTitle(raw) {
  if (!raw) return ''
  // 去掉末尾的 " | XXX"、" — XXX"、" - XXX"、" · XXX"
  return raw
    .replace(/\s[\|\—\-·]\s[^|\—\-·]+$/, '')
    .trim()
}
```

在 `fetchArticleFromUrl` 返回 title 时应用：

```js
title: cleanTitle(article.title) || new URL(normalized).hostname,
```

## 方案三：img src 相对路径转绝对

### 原理

`<base href>` 只影响浏览器渲染，不影响 DOM 属性值。TurndownService 使用 `getAttribute('src')` 读取原始值，导致相对路径被原样写入 Markdown。

修复方式：在 `parseArticle` 中，Readability 解析完成后、调用 `htmlToMarkdown` 之前，遍历 `article.content` 的 DOM，将所有 `<img>` 的 `src` 替换为绝对 URL。

### 实现

```js
function resolveImageUrls(htmlString, baseUrl) {
  const parser = new DOMParser()
  const doc = parser.parseFromString(htmlString, 'text/html')
  doc.querySelectorAll('img[src]').forEach((img) => {
    try {
      img.setAttribute('src', new URL(img.getAttribute('src'), baseUrl).href)
    } catch {
      // 无效 URL，保持原值
    }
  })
  return doc.body.innerHTML
}
```

在 `parseArticle` 的返回值链路中：

```js
const resolvedContent = resolveImageUrls(article.content ?? '', normalized)
return {
  ...article,
  resolvedContent,
}
// 调用方改为 htmlToMarkdown(article.resolvedContent)
```

## 方案四：自建 Netlify Function 代理

### 原理

用 Netlify Function 在服务器端发起 HTTP 请求，规避浏览器 CORS 限制，同时可设置 `User-Agent`、`Accept-Language` 等 header，提高成功率。

### 接口设计

```
GET /.netlify/functions/proxy?url=<encoded_url>
Response: { html: string }
```

### 实现

**`netlify/functions/proxy.js`**：

```js
export default async (req) => {
  const url = new URL(req.url).searchParams.get('url')
  if (!url) return new Response(JSON.stringify({ error: 'missing url' }), { status: 400 })

  try {
    const res = await fetch(url, {
      headers: {
        'User-Agent': 'Mozilla/5.0 (compatible; ReadRead/1.0)',
        'Accept': 'text/html,application/xhtml+xml',
        'Accept-Language': 'en-US,en;q=0.9',
      },
      signal: AbortSignal.timeout(20000),
    })
    if (!res.ok) return new Response(JSON.stringify({ error: `upstream ${res.status}` }), { status: 502 })
    const html = await res.text()
    return new Response(JSON.stringify({ html }), {
      headers: { 'Content-Type': 'application/json' },
    })
  } catch (e) {
    return new Response(JSON.stringify({ error: e.message }), { status: 502 })
  }
}

export const config = { path: '/api/proxy' }
```

### 代理优先级调整

将自建代理加入 `PROXIES` 数组，排在首位；第三方代理作为 fallback：

```js
const PROXIES = [
  {
    buildUrl: (u) => `/api/proxy?url=${encodeURIComponent(u)}`,
    extract: async (res) => { const d = await res.json(); return d.html ?? null },
  },
  // ... 原有两个第三方代理
]
```

### 注意事项

- 需在 `netlify.toml` 中确认 Functions 目录配置（当前已有 `netlify/functions/`，直接新建文件即可）
- 本地开发时 `netlify dev` 会自动挂载 Function，无需额外配置
- 部分站点（Cloudflare 保护、严格反爬）仍可能返回 403，这是合理的失败，保持现有错误提示即可
