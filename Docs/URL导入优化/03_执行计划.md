# URL 导入优化 — 执行计划

## 分组

### Group A：低成本改进（urlImport.js）
> 改动集中在单一文件，可一次性完成

- [ ] A1：`img src` 相对路径转绝对（`resolveImageUrls` 函数 + 调用链路调整）

- [ ] A3：请求超时（`fetchWithTimeout` 封装，15s/proxy）

### Group B：Netlify Function 代理
> 需新建 Function 文件，并调整前端代理列表

- [ ] B1：新建 `netlify/functions/proxy.js`

## 执行顺序

```
A1 → A2 → A3  （可连续执行，均在 urlImport.js）

```

## 验收标准

| 项目 | 验收方式 |
|---|---|
| A1 img 绝对化 | 导入含相对路径图片的文章，Reader 中图片正常显示 |
| A2 标题清洗 | 导入 Medium / Substack 文章，标题不含 " \| Site" 尾缀 |
| A3 请求超时 | 模拟代理无响应（断网或修改 URL），15s 内提示错误而非无限等待 |
| B1+B2 自建代理 | 导入成功率不低于使用第三方代理时；netlify dev 本地可正常调用 Function |
