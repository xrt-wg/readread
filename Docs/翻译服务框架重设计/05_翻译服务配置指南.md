# 翻译服务配置指南

- **适用版本**：翻译框架 v2（两阶段架构）
- **面向对象**：开发者

---

## 概览

翻译服务分为两条独立链路，均通过 `config/translation.js` 统一配置：

| 链路 | 触发时机 | 配置入口 |
|------|---------|---------|
| **直译**（弹窗） | 用户选中文本 | `directConfig.activeProvider` |
| **AI 精译**（收藏卡片） | 用户点击收藏 | `aiConfig.activePreset` |

---

## 一、直译服务（弹窗）

### 当前支持的 Provider

| Provider | 标识符 | 是否需要 Key | 调用方式 |
|---------|--------|------------|---------|
| MyMemory | `'myMemory'` | 否（免费公共 API） | 前端直连 |
| DeepL Free | `'deepl'` | 是 | Netlify Function 代理 |
| 有道翻译 | `'youdao'` | 是 | Netlify Function 代理 |

### 切换 Provider

编辑 `config/translation.js`：

```js
export const directConfig = {
  activeProvider: 'deepl', // 改为目标 provider 标识符
}
```

### 为 DeepL / 有道配置 API Key

需要在 **Netlify 环境变量**中添加对应 Key（不要写入代码）：

**DeepL**：
```
PRESET_DEEPL_API_KEY=your_deepl_api_key
```

**有道**：
```
PRESET_YOUDAO_APP_KEY=your_youdao_app_key
PRESET_YOUDAO_APP_SECRET=your_youdao_app_secret
```

> Netlify Dashboard → Site → Environment variables → Add variable

---

### 添加新的直译 Provider

以接入**谷歌翻译**为例，完整流程如下：

**Step 1**：在 `config/translation.js` 的注释中登记新标识符

```js
export const directConfig = {
  activeProvider: 'google', // 新增：'myMemory' | 'deepl' | 'youdao' | 'google'
}
```

**Step 2**：若该服务需要 Key，在 `netlify/functions/translate.js` 中实现调用函数

```js
async function callGoogle(text) {
  const apiKey = process.env.PRESET_GOOGLE_TRANSLATE_KEY
  if (!apiKey) throw new Error('Google Translate key not configured')
  // ... 调用 Google Cloud Translation API
  return translatedText
}
```

**Step 3**：在 `netlify/functions/translate.js` 的路由分支中注册：

```js
// 在 handler 的 provider 路由中添加：
if (provider === 'google') {
  result = await callGoogle(body.word)
}
```

**Step 4（若免费/无需 Key）**：也可在前端直接实现，在 `src/services/directTranslation/` 目录下新建文件（如 `google.js`），然后在 `index.js` 的 `translateDirect` 函数中添加分支：

```js
import { translate as googleTranslate } from './google'

// translateDirect 函数内：
if (provider === 'google') {
  result = await googleTranslate(text, signal)
}
```

## 二、AI 精译服务（收藏卡片）

### 当前支持的 Preset

Preset 在 `config/presetModels.json` 中定义，通过 Netlify Function 代理调用（Key 在服务端）：

```json
{
  "gemini-preset": {
    "label": "Google Gemini",
    "kind": "gemini",
    "defaultModel": "gemini-2.5-flash"
  },
  "deepseek-preset": {
    "label": "DeepSeek",
    "kind": "openai-compat",
    "defaultModel": "deepseek-chat"
  }
}
```

### 切换 AI Preset

编辑 `config/translation.js`：

```js
export const aiConfig = {
  activePreset: 'deepseek-preset', // 切换为 deepseek
}
```

### 更换 Preset 的默认模型

编辑 `config/presetModels.json`：

```json
{
  "gemini-preset": {
    "defaultModel": "gemini-2.0-flash"  // 改为目标模型名
  }
}
```

### 为 Preset 配置 API Key

在 Netlify 环境变量中配置：

| Preset | 环境变量名 |
|--------|-----------|
| `gemini-preset` | `PRESET_GEMINI_API_KEY` |
| `deepseek-preset` | `PRESET_DEEPSEEK_API_KEY` |

---

### 添加新的 AI Preset

以接入 **Kimi（Moonshot）** 为例：

**Step 1**：在 `config/presetModels.json` 中注册：

```json
{
  "kimi-preset": {
    "label": "Kimi (Moonshot)",
    "kind": "openai-compat",
    "defaultModel": "moonshot-v1-8k"
  }
}
```

**Step 2**：在 `netlify/functions/translate.js` 中实现调用函数，OpenAI 兼容接口可复用已有模板：

```js
async function callKimi(prompt, maxTokens, model) {
  const apiKey = process.env.PRESET_KIMI_API_KEY
  if (!apiKey) throw new Error('Kimi preset key not configured on server')
  const res = await fetch('https://api.moonshot.cn/v1/chat/completions', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${apiKey}` },
    body: JSON.stringify({
      model,
      messages: [{ role: 'user', content: prompt }],
      max_tokens: maxTokens,
      temperature: 0.2,
    }),
  })
  // ... 处理响应
}
```

**Step 3**：在 Netlify Function 的 provider 路由中注册：

```js
if (provider === 'kimi-preset') {
  const model = resolvePresetModel(provider)
  result = await callKimi(body.prompt, body.maxTokens, model)
}
```

**Step 4**：在 `config/translation.js` 中切换：

```js
export const aiConfig = {
  activePreset: 'kimi-preset',
}
```

**Step 5**：在 Netlify 添加环境变量 `PRESET_KIMI_API_KEY`。

## 三、修改 AI Prompt

所有 Prompt 集中在 `config/translation.js` 的 `prompts` 对象中：

```js
export const prompts = {
  // 词/短句收藏卡片，返回 JSON {meaning, contextTranslation}
  bookmarkCard: (word, context) => `...`,

  // 句子收藏卡片，返回中文字符串
  sentence: (text) => `...`,

  // 段落收藏卡片，返回中文字符串
  paragraph: (text) => `...`,
}
```

**注意事项**：
- `bookmarkCard` 必须要求 AI 返回严格 JSON `{"meaning":"...","contextTranslation":"..."}`，不要删除此约束
- `sentence` / `paragraph` 返回纯字符串，只需要中文翻译，不要额外说明
- 修改 Prompt 后无需重新部署前端，但若修改了返回格式，需同步修改 `src/services/aiProviders/index.js` 中的 `parseWordPhraseBundle` 函数

## 四、修改 Token 上限

```js
export const tokenLimits = {
  bookmarkCard: 260,   // 词/短句卡片（JSON 格式，含两个字段）
  sentence: 200,       // 句子翻译
  paragraph: 600,      // 段落翻译
}
```

Token 上限影响 AI 输出长度，过低可能导致截断。当前值在实测中足够，如遇长段落截断可适当增大 `paragraph`。

## 五、本地开发调试

直译和 AI 精译均依赖 Netlify Function，本地运行时需使用 Netlify CLI：

```bash
# 安装 Netlify CLI（若未安装）
npm install -g netlify-cli

# 启动本地开发（同时启动 Vite 和 Netlify Functions）
netlify dev
```

本地 Netlify Function 会从项目根目录的 `.env` 文件读取环境变量：

```
# .env（不要提交到 Git）
PRESET_GEMINI_API_KEY=your_key_here
PRESET_DEEPSEEK_API_KEY=your_key_here
```

> `.env` 已在 `.gitignore` 中，请确认不会被提交。
