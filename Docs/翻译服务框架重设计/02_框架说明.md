# 翻译服务框架说明（新）

- **版本**：v1.0
- **日期**：2026-04-01
- **状态**：设计完成，待实施

---

## 设计目标

1. 弹窗翻译极速响应，不打断阅读节奏
2. 收藏卡片高质量 AI 精译，后台异步，用户无感知等待
3. 用户零配置，隐藏所有翻译服务细节
4. 开发者可通过单一配置文件切换服务、调整 Prompt

---

## 整体架构

```
用户划词
  │
  └─ 弹窗触发 ──────────────────────────────► 直译服务（极速）
                                               └─ 返回：选中文本译文
                                               └─ 不含语境句翻译

用户点击收藏
  │
  ├─ 立即写入 bookmark（含原文，翻译字段为 null）
  ├─ 侧边栏可即时显示占位卡片（loading 状态）
  └─ 后台触发 AI 翻译 ───────────────────────► AI 精译服务
                                               └─ 返回：{meaning, contextTranslation}
                                               └─ 异步更新 bookmark 翻译字段
                                               └─ 侧边栏卡片自动更新
```

---

## 两阶段说明

### 阶段一：弹窗直译

| 属性 | 说明 |
|------|------|
| 触发时机 | 用户划词释放鼠标 |
| 翻译内容 | 选中文本译文（词、短句、句子、段落均可） |
| 语境句翻译 | **不包含** |
| 翻译服务 | 直译服务（由 `config/translation.js` 的 `directConfig.activeProvider` 决定） |
| 响应要求 | 极速，目标 < 500ms |

### 阶段二：收藏卡片 AI 精译

| 属性 | 说明 |
|------|------|
| 触发时机 | 用户点击收藏按钮 |
| 翻译内容 | `{meaning, contextTranslation}`（词义 + 语境句翻译） |
| 翻译服务 | AI 服务（由 `config/translation.js` 的 `aiConfig.activePreset` 决定） |
| 响应要求 | 后台异步，不阻塞用户操作 |
| 未就绪状态 | bookmark 数据含原文，翻译字段为 `null`，卡片显示 loading |

## 模块职责

### `config/translation.js`（新建）

**唯一开发者配置入口**，集中管理：

```js
// 直译服务配置
export const directConfig = {
  activeProvider: 'myMemory',  // 切换此处即可换直译服务
}

// AI 精译配置
export const aiConfig = {
  activePreset: 'gemini-preset',  // 切换此处即可换 AI 服务
}

// Prompt 模板（修改此处调整翻译指令）
export const prompts = {
  bookmarkCard: (word, context) =>
    `You are a translation assistant.
Translate according to the context and return STRICT JSON only (no markdown, no explanation):
{"meaning":"...","contextTranslation":"..."}

Rules:
- "meaning": concise Chinese meaning of "${word}" in this context, <= 12 Chinese characters.
- "contextTranslation": Chinese translation of this sentence: "${context}".
- Keep wording natural and accurate.`,

  sentence: (text) =>
    `Translate the following English sentence to Chinese. Reply with ONLY the Chinese translation, no explanation:\n\n${text}`,

  paragraph: (text) =>
    `Translate the following English text to Chinese. Reply with ONLY the Chinese translation, no explanation:\n\n${text}`,
}

// Token 限制
export const tokenLimits = {
  bookmarkCard: 260,
  sentence: 200,
  paragraph: 600,
}
```

### Hook 层（重构后）

| Hook | 职责 | 输出 |
|------|------|------|
| `useDirectTranslation` | 弹窗直译，接收任意文本 | `{result: string, loading, error}` |
| `useBookmarkAI` | 收藏触发 AI 精译，异步写回 bookmark | 内部状态，不向弹窗暴露 |

> 移除：`useAIWordTranslation`、`useDirectWordTranslation`（合并/替换）

### 服务层

#### `services/directTranslation/index.js`

职责不变，入口函数 `translateDirect(text, provider, signal)` 签名略调整（去掉 `contextSentence` 参数，弹窗不再需要）。

支持 provider：`myMemory`、`deepl`、`youdao`（后续扩展 `google`）。

#### `services/aiProviders/index.js`

职责不变，但 `buildPrompt` 和 `getMaxTokens` 改为从 `config/translation.js` 读取，不再硬编码。

#### `netlify/functions/translate.js`

无需改动，继续作为 preset AI 和需要 Key 的直译服务（DeepL/有道）的服务端代理。

### 数据层

#### Bookmark 数据结构扩展

新增翻译状态字段：

```js
{
  id: '...',
  text: '...',           // 原文（收藏时立即写入）
  context: '...',        // 语境句原文（收藏时立即写入）
  translation: null,     // AI 翻译结果，null 表示待生成
  // translation 就绪后：{ meaning: '...', contextTranslation: '...' }
  translationStatus: 'pending' | 'done' | 'error',
  // ...其他已有字段
}
```

### 组件层

#### `TranslationPopup`

**简化**：移除所有 AI 相关 props（`aiResult`、`aiLoading`、`aiError`、`ctxResult`、`ctxLoading`），只保留：

```
selectedText / selectionType / position
result / loading / error        ← 直译结果
contextSentence                 ← 原文语境句（仍显示，不翻译）
onClose / onBookmark / isBookmarked
```

#### `BookmarkPanel` / `BookmarkHoverCard`

新增对 `translationStatus === 'pending'` 的渲染：显示原文 + loading 占位，翻译就绪后自动更新。

#### `AISettingsPanel`

评估下线或仅作为开发者调试入口（不在用户侧暴露）。

## 与现有架构的差异

| 维度 | 现有架构 | 新架构 |
|------|---------|-------|
| 弹窗翻译路径 | AI（默认）或直译（可配置） | **仅直译** |
| 弹窗语境句翻译 | 有（AI bundle） | **无** |
| 收藏卡片翻译 | 收藏时使用弹窗已有结果 | **收藏触发独立 AI 请求** |
| 用户配置入口 | AISettingsPanel（provider / key / mode） | **无** |
| Prompt 位置 | 硬编码在 `services/aiProviders/index.js` | **集中在 `config/translation.js`** |
| Hook 数量 | 3 个（含 1 个残留） | 2 个（职责更清晰） |
